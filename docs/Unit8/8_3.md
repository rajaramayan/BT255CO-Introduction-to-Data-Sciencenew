# 8.3 Privacy Concerns and Regulations

---

## Theory

### Privacy in the Digital Age

!!! note "Definition"
    **Data Privacy** is the right of individuals to control how their personal information is collected, stored, used, and shared.

**Categories of personal data:**
- **Personally Identifiable Information (PII):** Name, address, phone, email, Aadhaar/SSN, passport number
- **Sensitive Personal Data:** Health records, biometrics, financial data, religion, sexual orientation
- **Pseudonymised Data:** Data linked to an identifier that can be re-linked to a person
- **Anonymised Data:** Data where the individual cannot be identified — GDPR does not apply

---

### Key Privacy Concerns

| Concern | Description | Example |
|---------|-------------|---------|
| **Data Breach** | Unauthorised access to personal data | 500M LinkedIn profiles leaked (2021) |
| **Re-identification** | Anonymised data traced back to individuals | Netflix ratings dataset de-anonymised by researchers |
| **Surveillance** | Tracking individuals via data collection | Facial recognition in public spaces |
| **Third-party sharing** | Data sold/shared without user knowledge | Facebook → Cambridge Analytica |
| **Profiling** | Building detailed personal profiles | Insurance companies using browsing history |
| **Data retention** | Keeping personal data longer than necessary | Old customer records not deleted |

---

### Major Data Privacy Regulations

=== "GDPR (EU)"
    **General Data Protection Regulation** — Effective May 2018

    | Aspect | Details |
    |--------|---------|
    | Jurisdiction | European Union + any entity processing EU citizens' data |
    | Key rights | Right to access, right to erasure ("right to be forgotten"), right to portability |
    | Consent | Must be freely given, specific, informed, and unambiguous |
    | Penalties | Up to **€20 million or 4% of global annual turnover** (whichever is higher) |
    | Data breach | Must notify regulators within **72 hours** |
    | Principles | Lawfulness, fairness, transparency, purpose limitation, data minimisation, accuracy, storage limitation, integrity, accountability |

=== "CCPA (USA)"
    **California Consumer Privacy Act** — Effective January 2020

    | Aspect | Details |
    |--------|---------|
    | Jurisdiction | Businesses serving California residents |
    | Key rights | Right to know, right to delete, right to opt out of sale |
    | Focus | Transparency about data collection and sale |

=== "DPDP (India)"
    **Digital Personal Data Protection Act** — 2023

    | Aspect | Details |
    |--------|---------|
    | Jurisdiction | Processing of personal data of Indian residents |
    | Key rights | Right to information, correction, erasure, grievance redressal |
    | Data Fiduciary | Entity that processes data must register and comply |
    | Penalties | Up to **₹250 crore** per violation |
    | Children's data | Enhanced protections; parental consent required |

---

### Privacy-Preserving Techniques

| Technique | Description | Use Case |
|-----------|-------------|---------|
| **Anonymisation** | Remove all PII permanently | Public dataset release |
| **Pseudonymisation** | Replace PII with a pseudonym (reversible) | Internal data processing |
| **Data Masking** | Obscure sensitive values (e.g., ****1234) | Show masked card numbers |
| **Differential Privacy** | Add statistical noise to outputs | Apple's keyboard usage analytics |
| **Federated Learning** | Train ML on distributed data without centralising it | Google Keyboard ML |
| **Encryption** | Encrypt data at rest and in transit | All sensitive storage and transmission |

---

## Summary

!!! success "Key Takeaways"
    - Data privacy is a fundamental right; regulations like GDPR, CCPA, and India's DPDP Act enforce it
    - **GDPR** has the highest global impact — it applies to any organisation processing EU citizens' data
    - India's **DPDP Act (2023)** is the national framework for data protection
    - Privacy-preserving techniques: anonymisation, differential privacy, federated learning, encryption
    - A **data breach** under GDPR must be reported within 72 hours
