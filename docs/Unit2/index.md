# Unit 2 — Data Munging
**Duration: 5 Hours**

---

## Overview

**Data Munging** (also called *Data Wrangling*) is the process of transforming and mapping raw, messy data into a format suitable for analysis. Real-world data is rarely clean — it contains duplicates, missing values, unit mismatches, and inconsistencies. This unit covers how to detect, handle, and prevent these issues.

## Quick Reference

| Term | Meaning |
|------|---------|
| Data Munging | Transforming raw/messy data into a clean, usable format |
| Data Quality | How well data meets accuracy, completeness, and consistency requirements |
| Duplicate | Two or more records referring to the same real-world entity |
| Missing Data | Values absent from the dataset (NaN, null, blank) |
| Nonstandard Data | Same data represented in different formats (e.g., dates, units) |
| Imputation | Estimating missing values using statistical or ML methods |
| Data Enrichment | Adding more value to data through correction or augmentation |
| Data Validation | Checking that data conforms to defined rules and constraints |
