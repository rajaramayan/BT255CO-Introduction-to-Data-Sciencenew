# 2.2 Common Issues with Real-World Data

---

## Theory

Real-world datasets are almost never perfect. The four most common data problems are: **Duplicates**, **Missing Data**, **Nonstandard Data**, and **Unit Mismatch**.

---

### 1. Duplicates

!!! note "Definition"
    A **duplicate** is a record that represents the same real-world entity more than once. Duplicates inflate counts, distort averages, and bias ML models.

**Causes:**
- Manual data entry errors
- Merging datasets from multiple sources
- System sync failures

**Types:**
| Type | Description | Example |
|------|-------------|---------|
| **Exact Duplicate** | All fields are identical | Same customer row entered twice |
| **Partial Duplicate** | Key fields match but minor differences exist | "Alice Smith" vs "alice smith" |
| **Logical Duplicate** | Different records represent the same entity | Customer ID 101 and 102 both refer to Alice |

---

### 2. Missing Data

!!! note "Definition"
    **Missing data** refers to values that are absent from the dataset. In Pandas, missing values are represented as `NaN` (Not a Number) or `None`.

**Types of Missingness:**

| Type | Description | Example |
|------|-------------|---------|
| **MCAR** (Missing Completely At Random) | Absence is unrelated to any variable | Random sensor dropout |
| **MAR** (Missing At Random) | Absence depends on another observed variable | Men less likely to report weight |
| **MNAR** (Missing Not At Random) | Absence depends on the missing value itself | High earners refuse to state salary |

**Strategies:**
- **Drop** — remove rows/columns (when missingness > 30% or MCAR)
- **Impute with mean/median/mode** — simple, but distorts distributions
- **KNN Imputation** — fills based on similar records
- **Forward/Backward Fill** — for time series data

---

### 3. Nonstandard Data

!!! note "Definition"
    **Nonstandard data** means the same information is represented in multiple different formats within a dataset.

| Category | Nonstandard Examples | Standard Form |
|----------|---------------------|---------------|
| **Dates** | "01/12/2024", "December 1, 2024", "2024-12-01" | ISO 8601: "2024-12-01" |
| **Text case** | "Delhi", "DELHI", "delhi" | "Delhi" (title case) |
| **Boolean** | "Yes", "y", "1", "TRUE" | `True` / `False` |
| **Phone numbers** | "+91-9876543210", "9876543210", "987-654-3210" | "+919876543210" |
| **Currency** | "₹50,000", "$50000", "50000 INR" | `50000` (float, standardised currency) |

---

### 4. Unit Mismatch

!!! note "Definition"
    **Unit mismatch** occurs when the same physical quantity is recorded in different units across records or columns.

| Column | Value 1 | Value 2 | Problem |
|--------|---------|---------|---------|
| `temperature` | 98.6 (°F) | 37.0 (°C) | Mixed units — model interprets both as the same scale |
| `weight` | 70 (kg) | 154 (lbs) | Same person's weight looks doubled |
| `distance` | 5.0 (km) | 3.1 (miles) | Different scales treated as same |

### Python Program — Detecting and Fixing Common Issues

```python linenums="1" title="real_world_issues.py"
# Program : Common Real-World Data Issues
# Topic   : 2.2 Common Issues with Real-World Data
# Author  : BT255CO Lecture Notes

import pandas as pd
import numpy as np

# -------------------------------------------------------
# Simulated messy dataset
# -------------------------------------------------------
data = {
    "id":          [1, 2, 3, 2, 4, 5],
    "name":        ["Alice", "Bob", "Carol", "Bob", "David", "Eva"],
    "dob":         ["1998-05-14", "15/06/1997", "March 3, 2000",
                    "15/06/1997", "1999-11-22", "2001-01-30"],
    "weight_unit": ["kg", "lbs", "kg", "lbs", "kg", "kg"],
    "weight":      [55.0, 154.0, 62.0, 154.0, 70.0, 48.0],
    "enrolled":    ["Yes", "yes", "1", "yes", "TRUE", "No"],
    "score":       [85, None, 90, None, 75, 88],
}

df = pd.DataFrame(data)
print("Original Messy Data:")
print(df)
print()

# =========================================================
# Issue 1: DUPLICATES
# =========================================================
print("─" * 50)
print("Issue 1: Duplicates")
dups = df[df.duplicated(subset=["id"])]
print(f"Duplicate IDs found: {dups['id'].tolist()}")
df = df.drop_duplicates(subset=["id"], keep="first")
print(f"After dedup: {len(df)} rows\n")

# =========================================================
# Issue 2: MISSING DATA
# =========================================================
print("─" * 50)
print("Issue 2: Missing Data")
print(f"Missing scores: {df['score'].isnull().sum()}")
df["score"].fillna(df["score"].median(), inplace=True)
print(f"After imputation with median ({df['score'].median()}): no missing values.\n")

# =========================================================
# Issue 3: NONSTANDARD DATA — dates
# =========================================================
print("─" * 50)
print("Issue 3: Nonstandard Dates")
print(f"Before: {df['dob'].unique()}")

df["dob_clean"] = pd.to_datetime(df["dob"], dayfirst=False, errors="coerce",
                                  infer_datetime_format=True)
print(f"After (ISO 8601):\n{df['dob_clean']}\n")

# Nonstandard booleans
print("Nonstandard Boolean (enrolled):")
print(f"  Before: {df['enrolled'].unique()}")
bool_map = {"yes": True, "1": True, "true": True, "no": False, "0": False}
df["enrolled_clean"] = df["enrolled"].str.lower().map(bool_map)
print(f"  After:  {df['enrolled_clean'].tolist()}\n")

# =========================================================
# Issue 4: UNIT MISMATCH — convert lbs to kg
# =========================================================
print("─" * 50)
print("Issue 4: Unit Mismatch (weight)")
print(f"  Before:\n{df[['name', 'weight', 'weight_unit']]}")

def to_kg(row):
    if row["weight_unit"] == "lbs":
        return round(row["weight"] * 0.453592, 1)
    return row["weight"]

df["weight_kg"] = df.apply(to_kg, axis=1)
print(f"\n  After (all in kg):\n{df[['name', 'weight', 'weight_unit', 'weight_kg']]}")
```

## Summary

!!! success "Key Takeaways"
    - The four main real-world data issues are **Duplicates, Missing Data, Nonstandard Data, and Unit Mismatch**
    - Duplicates are removed with `drop_duplicates()`; missing data with `fillna()` or `dropna()`
    - `pd.to_datetime()` handles multiple date formats automatically
    - Unit mismatches require domain knowledge and a conversion function applied row-by-row
    - Always **document** data cleaning decisions for reproducibility
