# 1.3 Limitations of Data Science

---

## Theory

Despite its power, Data Science has significant limitations that practitioners must be aware of. Ignoring these can lead to poor decisions, biased outcomes, or harmful consequences.

---

### 1. Data Quality Issues

!!! warning "Garbage In, Garbage Out (GIGO)"
    The quality of a model's output is directly limited by the quality of its input data.
    No algorithm can fix fundamentally flawed data.

| Issue | Description | Example |
|-------|-------------|---------|
| **Missing Values** | Incomplete records reduce model reliability | Survey respondents skip income field |
| **Noisy Data** | Errors and random variation distort patterns | Sensor malfunctions record wrong temperature |
| **Outliers** | Extreme values can skew statistics | One billionaire in a salary dataset inflates the mean |
| **Duplicates** | Repeated records bias results | Same customer appears twice in churn dataset |
| **Inconsistent Format** | Different representations of the same data | Dates as "01/12/2024", "2024-12-01", "Dec 1, 2024" |

---

### 2. Data Privacy and Security

- Collecting personal data without consent violates regulations (GDPR, IT Act)
- Re-identification attacks can expose anonymised individuals
- Model inversion attacks can extract training data from deployed models
- Data breaches expose sensitive personal and financial information

---

### 3. Bias in Data and Models

!!! danger "Algorithmic Bias"
    When training data reflects historical inequalities or societal biases, 
    the model **perpetuates and amplifies** those biases.

| Type of Bias | Description | Example |
|-------------|-------------|---------|
| **Sampling Bias** | Training data is not representative | Facial recognition trained mostly on light-skinned faces performs poorly on dark-skinned faces |
| **Label Bias** | Human annotators introduce subjective labels | NLP toxic comment detectors flag African American dialect as more "toxic" |
| **Confirmation Bias** | Analysts unconsciously select data that confirms expectations | — |
| **Historical Bias** | Data reflects past discrimination | Loan models deny credit to groups historically denied loans |

---

### 4. Interpretability vs. Accuracy Trade-off

| Model Type | Accuracy | Interpretability |
|-----------|----------|-----------------|
| Linear Regression | Medium | High (coefficients are intuitive) |
| Decision Tree | Medium | High (visual flowchart) |
| Random Forest | High | Low (ensemble of 100+ trees) |
| Deep Neural Network | Very High | Very Low ("Black Box") |

!!! note "The Black Box Problem"
    A doctor cannot use a model that says "the patient will develop diabetes" without 
    being able to explain *why*. This limits adoption of complex models in critical fields.

---

### 5. Correlation ≠ Causation

One of the most common errors in Data Science is confusing correlation with causation.

> *"Ice cream sales and drowning deaths are highly correlated."*  
> Does this mean ice cream causes drowning? **No** — both are caused by hot weather (a confounding variable).

---

### 6. Scalability and Infrastructure

- Large datasets require expensive cloud computing and storage
- Real-time processing requires specialised infrastructure (Kafka, Spark)
- Training deep learning models needs GPU clusters
- Not all organisations have the budget or talent

---

### 7. Other Limitations

| Limitation | Description |
|-----------|-------------|
| **Overfitting** | Model memorises training data and fails on new data |
| **Underfitting** | Model is too simple to capture real patterns |
| **Data Scarcity** | Rare events (e.g., rare disease) have too few examples to train on |
| **Concept Drift** | Real-world patterns change over time; models become outdated |
| **Ethical Issues** | Automated decisions affecting people (hiring, bail) need oversight |

### Python Program — Demonstrating Data Quality Issues

```python linenums="1" title="data_limitations.py"
# Program : Demonstrating Data Limitations
# Topic   : 1.3 Limitations of Data Science
# Author  : BT255CO Lecture Notes

import pandas as pd
import numpy as np

# -------------------------------------------------------
# Simulated messy real-world dataset
# -------------------------------------------------------
data = {
    "name":   ["Alice", "Bob", "Carol", "Alice", None, "Eva"],
    "age":    [25, 200, 30, 25, 28, -5],         # 200 and -5 are outliers
    "salary": [50000, None, 60000, 50000, 45000, 55000],
    "city":   ["Delhi", "mumbai", "DELHI", "Delhi", "Mumbai", "chennai"],
}

df = pd.DataFrame(data)
print("Original (Messy) Dataset:")
print(df)
print()

# -------------------------------------------------------
# Issue 1: Missing values
# -------------------------------------------------------
print("Missing values per column:")
print(df.isnull().sum())
print()

# -------------------------------------------------------
# Issue 2: Duplicate rows
# -------------------------------------------------------
print(f"Number of duplicate rows: {df.duplicated().sum()}")
print()

# -------------------------------------------------------
# Issue 3: Outliers in age (valid range: 18–100)
# -------------------------------------------------------
invalid_ages = df[(df["age"] < 18) | (df["age"] > 100)]
print("Rows with invalid age values:")
print(invalid_ages)
print()

# -------------------------------------------------------
# Issue 4: Inconsistent text formatting in 'city'
# -------------------------------------------------------
print("Unique city values (before normalisation):", df["city"].unique())

# Normalise to title case
df["city"] = df["city"].str.title()
print("Unique city values (after normalisation):", df["city"].unique())
print()

# -------------------------------------------------------
# How bias can appear — mean salary with vs without outlier
# -------------------------------------------------------
salaries_with_outlier    = [50000, 60000, 45000, 55000, 10000000]
salaries_without_outlier = [50000, 60000, 45000, 55000]

mean_with    = np.mean(salaries_with_outlier)
mean_without = np.mean(salaries_without_outlier)

print(f"Mean salary WITH outlier:    ₹{mean_with:,.0f}")
print(f"Mean salary WITHOUT outlier: ₹{mean_without:,.0f}")
print("→ A single outlier inflated the mean by "
      f"₹{mean_with - mean_without:,.0f}!")
```

## Summary

!!! success "Key Takeaways"
    - Data Science is limited by **data quality** — missing values, noise, duplicates, and outliers
    - **Algorithmic bias** can perpetuate societal inequalities if training data is unrepresentative
    - **Correlation ≠ Causation** — a critical distinction for drawing valid conclusions
    - Complex models like deep networks are **difficult to interpret** ("black box problem")
    - **Concept drift** means models can become outdated as real-world patterns change
