# 7.6 Practices and Use Cases of Big Data

---

## Theory

Big Data is transforming every major industry. Below are the most impactful real-world applications and the practices used to realise them.

---

### Best Practices

| Practice | Description |
|----------|-------------|
| **Data Governance** | Define ownership, quality standards, and access policies |
| **Metadata Management** | Document what every dataset contains and its lineage |
| **Data Cataloguing** | Maintain a searchable index of available datasets |
| **Schema on Read** | Store raw data first; apply schema when reading (Data Lake approach) |
| **Columnar Storage** | Use Parquet/ORC for analytics (faster column scans) |
| **Data Partitioning** | Split large datasets by date/region for faster queries |
| **Monitoring & Alerting** | Detect pipeline failures and data drift automatically |

---

### MapReduce — The Core Processing Pattern

```python linenums="1" title="mapreduce_simulation.py"
# Program : MapReduce Pattern Simulation in Python
# Topic   : 7.6 Big Data Practices
# Author  : BT255CO Lecture Notes

from collections import defaultdict
from functools import reduce

# -------------------------------------------------------
# Simulated log data (e.g., web server access logs)
# -------------------------------------------------------
log_data = [
    "2024-01-01 GET /home 200",
    "2024-01-01 GET /products 200",
    "2024-01-01 GET /home 200",
    "2024-01-01 GET /checkout 404",
    "2024-01-02 GET /home 200",
    "2024-01-02 GET /products 200",
    "2024-01-02 GET /cart 200",
    "2024-01-02 GET /home 500",
    "2024-01-03 GET /home 200",
    "2024-01-03 GET /checkout 200",
]

# =========================================================
# MAP phase: extract (key, value) pairs
# =========================================================
def mapper(log_line):
    """Extract (page, 1) from each log entry."""
    parts = log_line.split()
    page  = parts[2]   # e.g., /home
    return (page, 1)

mapped = [mapper(line) for line in log_data]
print("MAP output:")
for item in mapped:
    print(f"  {item}")
print()

# =========================================================
# SHUFFLE & SORT phase: group by key
# =========================================================
grouped = defaultdict(list)
for key, value in mapped:
    grouped[key].append(value)

print("SHUFFLE & SORT output (grouped by page):")
for key, values in grouped.items():
    print(f"  {key}: {values}")
print()

# =========================================================
# REDUCE phase: aggregate values per key
# =========================================================
def reducer(key, values):
    """Sum all counts for this page."""
    return (key, sum(values))

results = [reducer(key, values) for key, values in grouped.items()]
results.sort(key=lambda x: -x[1])   # sort by count descending

print("REDUCE output (page visit counts):")
print(f"{'Page':<15} {'Visits':>8}")
print("-" * 25)
for page, count in results:
    print(f"{page:<15} {count:>8}")
```

**Output:**
```
MAP output:
  ('/home', 1)
  ('/products', 1)
  ('/home', 1)
  ('/checkout', 1)
  ('/home', 1)
  ('/products', 1)
  ('/cart', 1)
  ('/home', 1)
  ('/home', 1)
  ('/checkout', 1)

SHUFFLE & SORT output (grouped by page):
  /home: [1, 1, 1, 1, 1]
  /products: [1, 1]
  /checkout: [1, 1]
  /cart: [1]

REDUCE output (page visit counts):
Page              Visits
-------------------------
/home                  5
/products              2
/checkout              2
/cart                  1
```

## Summary

!!! success "Key Takeaways"
    - Big Data practices include governance, metadata management, columnar storage, and partitioning
    - **MapReduce** is the fundamental Big Data processing pattern: Map → Shuffle → Reduce
    - E-commerce, healthcare, finance, and smart cities are the biggest adopters
    - Amazon, Google, Netflix, and Uber have built multi-billion-dollar businesses on Big Data
