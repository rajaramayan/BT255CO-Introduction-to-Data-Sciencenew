# 6.3.1 Clustering — K-Means and Hierarchical

---

## Theory

!!! note "Definition"
    **Clustering** is an unsupervised learning technique that groups data points so that points in the same cluster are **more similar** to each other than to points in other clusters, without using any predefined labels.

---

### K-Means Clustering

**Algorithm:**

1. Choose $k$ (number of clusters)
2. Randomly initialise $k$ centroids
3. Assign each point to the nearest centroid (using Euclidean distance)
4. Recompute each centroid as the mean of its assigned points
5. Repeat steps 3–4 until centroids no longer move (convergence)

**Objective:** Minimise **Within-Cluster Sum of Squares (WCSS / Inertia):**

$$
\text{WCSS} = \sum_{j=1}^{k} \sum_{x_i \in C_j} \|x_i - \mu_j\|^2
$$

**Choosing k — The Elbow Method:**

Plot WCSS vs. k; the optimal k is at the "elbow" where WCSS starts decreasing slowly.

**Limitations:**
- Must specify $k$ in advance
- Assumes spherical, equally-sized clusters
- Sensitive to initialisation and outliers

---

### Hierarchical Clustering

**Algorithm (Agglomerative — bottom-up):**

1. Start: each point is its own cluster ($n$ clusters)
2. Merge the two closest clusters
3. Repeat until all points are in one cluster
4. Cut the dendrogram at the desired level to obtain $k$ clusters

**Linkage criteria (how to measure distance between clusters):**

| Linkage | Method | Description |
|---------|--------|-------------|
| **Single** | min distance | Distance between closest points in two clusters |
| **Complete** | max distance | Distance between farthest points |
| **Average** | avg distance | Average pairwise distance |
| **Ward** | variance | Minimise within-cluster variance (default, most common) |

**Advantages over K-Means:**
- No need to specify $k$ in advance
- Produces a full hierarchy (dendrogram)
- Works with any distance metric

### Python Program — K-Means and Hierarchical Clustering

```python linenums="1" title="clustering.py"
# Program : K-Means and Hierarchical Clustering
# Topic   : 6.3.1 Clustering
# Author  : BT255CO Lecture Notes

import numpy as np
import pandas as pd
from sklearn.cluster import KMeans, AgglomerativeClustering
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

np.random.seed(42)

# -------------------------------------------------------
# Generate synthetic dataset (3 clusters)
# -------------------------------------------------------
X_raw, y_true = make_blobs(n_samples=300, centers=3,
                             cluster_std=1.5, random_state=42)
scaler  = StandardScaler()
X       = scaler.fit_transform(X_raw)

print(f"Dataset: {X.shape[0]} samples, {X.shape[1]} features\n")

# =========================================================
# PART 1: K-Means — Finding Optimal k (Elbow Method)
# =========================================================
print("=" * 55)
print("PART 1: K-Means — Elbow Method")
print("=" * 55)
print(f"{'k':<5} {'WCSS (Inertia)':<20} {'Silhouette'}")
print("-" * 40)

wcss_values = []
for k in range(2, 9):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X)
    sil = silhouette_score(X, km.labels_) if k > 1 else "-"
    wcss_values.append(km.inertia_)
    print(f"{k:<5} {km.inertia_:<20.2f} {sil:.3f if isinstance(sil, float) else sil}")
print()

# =========================================================
# PART 2: K-Means with Optimal k=3
# =========================================================
print("=" * 55)
print("PART 2: K-Means (k=3) Results")
print("=" * 55)

kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans.fit(X)
labels_km = kmeans.labels_

print(f"Inertia (WCSS)    : {kmeans.inertia_:.2f}")
print(f"Silhouette Score  : {silhouette_score(X, labels_km):.3f}")
print(f"Iterations        : {kmeans.n_iter_}")
print(f"\nCluster sizes:")
for c in range(3):
    count = (labels_km == c).sum()
    print(f"  Cluster {c}: {count} points")
print()

# =========================================================
# PART 3: Hierarchical Clustering
# =========================================================
print("=" * 55)
print("PART 3: Hierarchical (Agglomerative) Clustering")
print("=" * 55)

# Try different linkage methods
for linkage_method in ["ward", "complete", "average"]:
    hc = AgglomerativeClustering(n_clusters=3, linkage=linkage_method)
    labels_hc = hc.fit_predict(X)
    sil = silhouette_score(X, labels_hc)
    print(f"  Linkage={linkage_method:<10}: Silhouette = {sil:.3f}")

print()

# =========================================================
# PART 4: Real-world example — Customer Segmentation
# =========================================================
print("=" * 55)
print("PART 4: Customer Segmentation")
print("=" * 55)

np.random.seed(5)
customers = pd.DataFrame({
    "annual_income_k":  np.concatenate([
        np.random.normal(30, 5, 60),   # low income
        np.random.normal(70, 8, 80),   # mid income
        np.random.normal(110, 10, 60), # high income
    ]),
    "spending_score": np.concatenate([
        np.random.normal(40, 10, 60),  # moderate spenders
        np.random.normal(55, 12, 80),  # average spenders
        np.random.normal(75, 8, 60),   # high spenders
    ]),
})

X_cust = StandardScaler().fit_transform(customers)
km_cust = KMeans(n_clusters=3, random_state=42, n_init=10)
customers["segment"] = km_cust.fit_predict(X_cust)

print(customers.groupby("segment")[["annual_income_k", "spending_score"]].mean().round(1))
print()
labels_map = {0: "Budget Buyers", 1: "Average Spenders", 2: "Premium Customers"}
# (label assignment may vary by run)
print("Business segments identified by K-Means:")
for seg, grp in customers.groupby("segment"):
    print(f"  Segment {seg}: {len(grp)} customers  "
          f"| Avg income: ₹{grp['annual_income_k'].mean():.0f}k  "
          f"| Avg spending: {grp['spending_score'].mean():.0f}")
```

## Summary

!!! success "Key Takeaways"
    - **K-Means** is fast and scalable; needs $k$ specified; assumes spherical clusters
    - **Elbow Method** plots WCSS vs. k — choose k at the "elbow"
    - **Silhouette Score** measures cluster quality: values near 1.0 are best
    - **Hierarchical Clustering** builds a dendrogram; no need to pre-specify $k$; Ward linkage is most common
    - Always **scale features** before clustering
