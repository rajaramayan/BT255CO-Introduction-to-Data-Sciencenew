# 6.2.3 Support Vector Machines (SVM)

---

## Theory

!!! note "Definition"
    A **Support Vector Machine (SVM)** is a supervised learning algorithm that finds the **optimal hyperplane** that maximises the **margin** between two classes in the feature space.

### Key Concepts

**Hyperplane:** A decision boundary that separates two classes. In 2D it's a line; in 3D it's a plane; in n-dimensions it's a hyperplane.

**Margin:** The distance between the hyperplane and the nearest data points (support vectors) from each class.

**Support Vectors:** The data points closest to the decision boundary. Only these points determine the hyperplane position.

$$
\text{Margin} = \frac{2}{\|\mathbf{w}\|}
$$

The SVM optimisation problem:

$$
\min_{\mathbf{w}, b} \frac{1}{2}\|\mathbf{w}\|^2 \quad \text{subject to } y_i(\mathbf{w} \cdot x_i + b) \geq 1
$$

---

### Hard Margin vs. Soft Margin

| Type | When to use | C parameter |
|------|-------------|-------------|
| **Hard Margin** | Data is linearly separable; no outliers | N/A |
| **Soft Margin** | Data is not perfectly separable | C controls trade-off |

**Regularisation parameter C:**
- **Large C** → small margin, fewer misclassifications (risks overfitting)
- **Small C** → large margin, more misclassifications allowed (better generalisation)

---

### The Kernel Trick

When data is not linearly separable in the original space, the **kernel trick** maps it to a higher-dimensional space where it becomes separable — **without explicitly computing the transformation**.

| Kernel | Formula | Use Case |
|--------|---------|---------|
| **Linear** | $K(x_i, x_j) = x_i \cdot x_j$ | Linearly separable data |
| **Polynomial** | $K(x_i, x_j) = (x_i \cdot x_j + 1)^d$ | Polynomial boundaries |
| **RBF (Gaussian)** | $K(x_i, x_j) = e^{-\gamma\|x_i - x_j\|^2}$ | Most common; non-linear boundaries |
| **Sigmoid** | $K(x_i, x_j) = \tanh(\kappa x_i \cdot x_j + c)$ | Similar to neural networks |

### Python Program — SVM Classification

```python linenums="1" title="svm_demo.py"
# Program : Support Vector Machines (SVM)
# Topic   : 6.2.3 Support Vector Machines
# Author  : BT255CO Lecture Notes

import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import load_breast_cancer, make_circles
from sklearn.model_selection import train_test_split, GridSearchCV
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import accuracy_score, classification_report

np.random.seed(42)

# =========================================================
# PART 1: SVM for Classification (Breast Cancer Dataset)
# =========================================================
print("=" * 55)
print("PART 1: SVM — Breast Cancer Classification")
print("=" * 55)

data = load_breast_cancer()
X, y = data.data, data.target    # 569 samples, 30 features, 2 classes (benign/malignant)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y
)

# Scale (CRITICAL for SVM — sensitive to feature scale)
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

# Train with RBF kernel
svm_rbf = SVC(kernel="rbf", C=1.0, gamma="scale", probability=True)
svm_rbf.fit(X_train_s, y_train)
y_pred = svm_rbf.predict(X_test_s)

print(f"Dataset  : {X.shape[0]} samples, {X.shape[1]} features")
print(f"Classes  : {data.target_names.tolist()}")
print(f"Kernel   : RBF  |  C=1.0")
print(f"Accuracy : {accuracy_score(y_test, y_pred) * 100:.1f}%")
print(f"Support Vectors: {svm_rbf.n_support_} (per class)")
print()

# =========================================================
# PART 2: Comparing Kernels
# =========================================================
print("=" * 55)
print("PART 2: Kernel Comparison")
print("=" * 55)

kernels = ["linear", "poly", "rbf", "sigmoid"]
print(f"{'Kernel':<12} {'Accuracy':>10}")
print("-" * 24)
for k in kernels:
    svm_k = SVC(kernel=k, C=1.0, gamma="scale")
    svm_k.fit(X_train_s, y_train)
    acc = accuracy_score(y_test, svm_k.predict(X_test_s))
    print(f"{k:<12} {acc*100:>9.1f}%")
print()

# =========================================================
# PART 3: Non-linear data — Circles (requires RBF)
# =========================================================
print("=" * 55)
print("PART 3: Non-Linear Data (Concentric Circles)")
print("=" * 55)

X_c, y_c = make_circles(n_samples=300, noise=0.1, factor=0.4, random_state=42)
Xc_tr, Xc_te, yc_tr, yc_te = train_test_split(X_c, y_c, test_size=0.3)

sc2 = StandardScaler()
Xc_tr_s = sc2.fit_transform(Xc_tr)
Xc_te_s = sc2.transform(Xc_te)

# Linear SVM fails on circular data
svm_lin = SVC(kernel="linear").fit(Xc_tr_s, yc_tr)
# RBF SVM handles it
svm_rbf2 = SVC(kernel="rbf", C=1.0, gamma="scale").fit(Xc_tr_s, yc_tr)

print(f"Linear SVM on circles : {svm_lin.score(Xc_te_s, yc_te)*100:.1f}%  ← fails!")
print(f"RBF SVM on circles    : {svm_rbf2.score(Xc_te_s, yc_te)*100:.1f}%  ← succeeds!")
print()

# =========================================================
# PART 4: Hyperparameter Tuning with GridSearchCV
# =========================================================
print("=" * 55)
print("PART 4: Hyperparameter Tuning (GridSearchCV)")
print("=" * 55)

param_grid = {
    "C":     [0.1, 1, 10, 100],
    "gamma": ["scale", "auto", 0.01, 0.001],
    "kernel": ["rbf"],
}
grid_search = GridSearchCV(SVC(), param_grid, cv=5, scoring="accuracy", n_jobs=-1)
grid_search.fit(X_train_s, y_train)

best = grid_search.best_estimator_
best_acc = accuracy_score(y_test, best.predict(X_test_s))

print(f"Best parameters : {grid_search.best_params_}")
print(f"Best CV score   : {grid_search.best_score_*100:.1f}%")
print(f"Test accuracy   : {best_acc*100:.1f}%")
print()
print("Classification Report (Best SVM):")
print(classification_report(y_test, best.predict(X_test_s),
                              target_names=data.target_names))
```

## Summary

!!! success "Key Takeaways"
    - SVM finds the **maximum-margin hyperplane** between classes
    - **Support vectors** are the critical data points that define the decision boundary
    - The **kernel trick** enables SVM to handle non-linearly separable data (RBF is most popular)
    - **C** controls the margin-error trade-off; **gamma** controls the RBF kernel's influence radius
    - **Always scale** features before training SVM
    - Use **GridSearchCV** to find optimal C and gamma values
